# 🏺 The **Archivist's Vault** 💾

## 🛑 **STOP!** What Chu Talkin' **Bao**t?

This directory, aptly named **`assets-archive`** (or whatever your folder is called), is **not** part of the active project codebase. If you're looking for the final, deployed assets—you're in the wrong steamer basket!

---

## 🧭 **What Is This Place?** (It's **Unbelieva-Bao**!)

Think of this folder as the project's **time capsule** and **digital workshop floor**. This is a **non-essential**, but incredibly valuable, repository for all the **raw, unflattened, and high-fidelity source materials** that went into creating the project.

This vault ensures that we can always return to the original **source files** for future updates, redesigns, or simply to marvel at the journey. It's a place for all the little ingredients that make the final product so **Bao-tiful**.

---

## 🌟 **What's Inside?** (The **Filling**!)

This is where the magic (and the mess!) of creation lives. You'll find:

| Resource Type | Purpose & Examples |
| :--- | :--- |
| **🖼️ Raw Graphics** | Original, unflattened, and high-resolution source files. E.g., **Inkscape SVG** files, **Adobe Illustrator AI** files, layered **Photoshop PSDs**. |
| **📸 Original Media** | Source photography, untouched audio clips, or raw video files before any compression or cropping. |
| **📄 Source Documents** | Wireframes, concept sketches (as scans), mood boards, or original license documents for commercial assets. |
| **🎨 Unused Drafts** | Graphics, icons, or design concepts that didn't make the final cut but might be useful later. |

---

## 🚀 **How To Use This Folder**

1.  **Read-Only:** Treat this folder as **read-only** for general development. You shouldn't be modifying these files unless you are actively performing a major asset update.
2.  **Asset Generation:** If you need to modify a final asset (e.g., changing `logo.png`), you must first edit its source file here (e.g., `logo_source.svg`), and then **export** the final, optimized version back into the **active** project folder (e.g., `src/assets/`).
3.  **Future Proofing:** If you introduce a new resource to the project, its **raw, unoptimized source file** should be added to the appropriate sub-folder within this archive.
